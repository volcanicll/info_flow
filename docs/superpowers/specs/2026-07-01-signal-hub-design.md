# InfoFlow · 信号中枢（Signal Hub）特色功能设计

> 状态：待评审
> 创建：2026-07-01
> 范围：为 InfoFlow 建立头号差异化特色「信息即信号」，将资讯与市场异动实时双向关联
> 关联：本特色为应用「之一」且「最重要」的特色，置于最醒目入口；应用母体仍是「AI 信息聚合 App」

---

## 一、产品定位与差异化记忆点

### 1.1 一句话定位

> **用户在 InfoFlow 里看到的每一条信息，都带着它的「市场身份证」。**

把现存的「纯资讯流」和「纯行情/雷达」割裂状态，融合为一条「事件 ↔ 异动」实时双向关联的时间线。

### 1.2 三个用户可感知的记忆点

| # | 记忆点 | 用户向朋友介绍时会说 |
|---|---|---|
| 1 | **带行情的新闻** | 「每条新闻上直接显示它提到的币/股的实时涨跌和异动」 |
| 2 | **带叙事的行情** | 「每个市场异动旁边自动聚合解释它的相关新闻」 |
| 3 | **市场脉搏时间线** | 「首页不是文章列表，是新闻和市场反应交织的实时线」 |

### 1.3 竞品空白

| 类型 | 代表 | 缺什么 |
|---|---|---|
| 纯行情 | 同花顺 / TradingView / Coinglass | 没有「为什么涨跌」的叙事 |
| 纯资讯 | 华尔街见闻 / 金十 | 没有「这条新闻触发了什么行情」 |
| **InfoFlow** | 本方案 | **「事件 ↔ 异动」实时双向关联，信息即信号** |

### 1.4 目标用户

- 加密 / 美股 / A股投资者：需要快速判断「这条消息值不值得交易」
- 信息工作者（分析师、产品、运营）：需要追踪行业事件的市场影响
- 内容创作者：需要快速产出「事件 + 市场反应」素材

## 三、功能设计

### 3.1 引擎层

#### 3.1.1 Ticker Resolver（标的识别引擎）

**职责**：从文章标题 + 正文识别涉及的金融标的，输出 `tickers: List<TickerRef>`。

**实现路线（务实，不依赖模型训练）**：

1. 维护**标的词典**（本地 JSON，支持热更新），分 4 类：
   - 加密：BTC / ETH / SOL …（含中文别名：比特币、大饼、以太坊、以太）
   - 美股：AAPL / TSLA / NVDA …（含公司名：苹果、特斯拉、英伟达）
   - A股：贵州茅台（600519）/ 宁德时代（300750）…
   - 宏观 / 商品：黄金、美元指数（DXY）、原油、纳指（NDX）、10Y 美债
2. **规则匹配**：Aho-Corasick 多模式匹配，按出现次数 / 标题权重计命中
3. **AI 增强（可选回退）**：规则匹配为空但文章明显为财经类时，调用一次已有 `ai_service` 抽取标的

**TickerRef 数据结构**：

```dart
class TickerRef {
  final String symbol;      // 'ETH' / 'AAPL' / '600519'
  final AssetClass asset;   // crypto / usStock / cnStock / macro
  final int mentions;       // 出现次数，用于排序
  final bool inTitle;       // 是否出现在标题（权重更高）
}
```

**性能要求**：单篇文章识别 < 20ms（纯本地词典路径）。

#### 3.1.2 Signal Link（信号关联引擎）

**职责**：在时间窗内把「事件」和「异动」配对，给出 **关联强度 0–100**。

**数据源**：

- **事件源** = RSS 文章（带 ticker 标签）
- **异动源** = 已有加密雷达信号 + 新增美股/A股波动扫描

**关联强度公式（透明、可解释）**：

```
关联强度 = 时间邻近度(40%) × 标的匹配度(30%) × 异动剧烈度(30%)

• 时间邻近度：事件与异动时间差越小越高；±2h 内有效，>2h 记 0
• 标的匹配度：精确标的命中 1.0；同板块（如 ETH 新闻 ↔ BTC 异动）0.5
• 异动剧烈度：复用雷达已有的 OI / 资金池 / 价格变化幅度归一化
```

**阈值**：

| 关联强度 | 处理 |
|---|---|
| ≥ 85 | 强信号：推送 + 红点呼吸 + 触觉反馈 |
| 70–84 | 中信号：进入脉搏时间线，不打扰 |
| < 70 | 弱关联：仅在 Ticker Lens 内可见 |

**可解释性**：每条配对保存打分明细，用户点击「为什么关联」可查看三项分数。

### 3.2 入口层

#### 3.2.1 市场脉搏时间线（Market Pulse）—— 首页核心

`/market` 路由内容替换为脉搏时间线（原 Dashboard 的 `_MarketGrid` 降级为顶部迷你行情条）。

时间线由三类卡片交织：

- **事件卡**（来自 RSS，左侧蓝条）：含标的徽章 + 关联异动条 + 关联强度
- **异动卡**（来自雷达/行情，左侧紫条）：含反向关联的新闻列表
- **预警卡**（强度 ≥85，左侧红条）：强信号提醒

顶部元素：

- `🔴 实时` 红点（呼吸动画，仅强度 ≥85 时亮）
- 迷你行情条（横向滚动，3–5 个关注标的）
- 快讯条（自动滚动，8 秒/条，仅展示最新 5 条标题）

#### 3.2.2 增强文章卡（Ticker 徽章）

升级现有 `article_card.dart`，按页面区分强弱：

| 页面 | 文章卡展示 |
|---|---|
| Feed（信息流） | 仅标的**标签**（`#黄金 #纳指`），不显示实时数字，保持阅读纯净 |
| Pulse（脉搏） | 完整行情徽章（标的 + 实时涨跌）+ 关联强度条 |

#### 3.2.3 标的看板 Ticker Lens —— 深度入口

从任意标的徽章进入，一屏汇聚：

- 大字号实时价格 + 24h 迷你 K 线
- 实时异动（复用加密雷达）
- 相关叙事（按关联强度降序的新闻列表）
- AI 解读（复用 `ai_service`，按需生成）

#### 3.2.4 阅读器实时行情条

`reader_page.dart` 顶部增加固定条：本文涉及标的的实时行情，阅读全程可见。

### 3.3 与现有模块的关系

| 现有模块 | 角色 | 改动 |
|---|---|---|
| 加密雷达 `crypto_radar` | 异动源 | 输出信号给 Signal Link；保留独立页 |
| 贵金属 `precious_metals` | 标的数据源 | 接入 Ticker Lens，作为「商品」类标的 |
| RSS feed | 事件源 | 文章卡增加 ticker 徽章 + 关联异动条 |
| Reader | 沉浸阅读 | 顶部加固定「本文涉及标的」行情条 |
| AI chat | 深度问答 | 增加「分析这个标的」快捷指令 |
| 搜索 `search` | 发现 | 支持按 ticker 搜索，结果带行情概览 |
| Dashboard | — | 内容替换为脉搏时间线，迷你行情条保留 |

### 3.4 新增市场数据源

**原则**：免费 / 免 Key 起步，复用现有 `dio` + `html/xml` 技术栈，不引入新依赖。

| 标的类 | 数据源 | 备注 |
|---|---|---|
| 加密 | Binance API（已有） | 实时 |
| 美股 | stooq.com（CSV，延迟 15min）或 Yahoo 非官方接口 | 延迟可接受 |
| A股 | 新浪财经 `hq.sinajs.cn` | 免费、实时 |
| 宏观/商品 | 扩展现有贵金属模块 | 黄金 / 原油 / DXY |

### 4.4 标志性交互

| # | 交互 | 行为 |
|---|---|---|
| 1 | 长按文章卡 | 弹半屏面板，实时显示本文发布后涉及标的的市场反应 |
| 2 | 下拉行情徽章 | 拉出解释该异动的 3 条最相关新闻，无需跳转 |
| 3 | 双指捏合 / 张开（脉搏页） | 捏合=只看强信号（≥70）；张开=看全部事件流 |
| 4 | 横滑事件卡 | 时间线过滤为该标的相关条目，顶部显示 `× ETH 过滤中` |
| 5 | 红点呼吸 + 触觉反馈 | 强度 ≥85 新信号时触发，制造「有事发生」紧迫感，不做无意义骚扰 |

### 4.5 视觉语言

| 维度 | 设定 | 理由 |
|---|---|---|
| 主色 | 深空蓝 `#0B1F3A`（深色主体）+ 荧光青 `#00E5FF`（信号高亮） | 区别同花顺红绿、Coinglass 紫，专业终端感 |
| 涨跌色 | 涨=暖橙 `#FF9500`，跌=冷青 `#00C2FF`（**不用红绿**） | 色盲友好 + 视觉独特，对标 Robinhood |
| 异动徽章 | 半透明描边胶囊 + 闪烁动画 | 终端警报感 |
| 关联强度条 | 渐变进度条（青→橙→红），高分带 🔥 | 一眼判断信号强度 |
| 数字字体 | 等宽（SF Mono / JetBrains Mono） | 金融数字对齐 |
| 背景 | 深色模式轻微噪点纹理 + 卡片 1px 描边 | 高级感，区别扁平 Material |

## 六、风险与对策

| 风险 | 对策 |
|---|---|
| 标的识别误判（如「苹果」指水果） | 标题优先 + 财经上下文二次确认；P0 先覆盖高确定性标的 |
| 关联强度公式不准 | 公式透明可调，初版权重写在配置；用户可反馈「关联错误」 |
| 第三方行情接口不稳定 | 统一封装 repository，支持多源降级；失败时仅展示标签不展示数字 |
| 信息过载（脉搏噪音大） | 默认只展示强度 ≥50 的条目；捏合手势切密度 |
| 推送骚扰 | 仅强度 ≥85 触发；用户可在设置中调阈值或关闭 |

---

## 七、不做的事（YAGNI）

明确以下内容**不在本特色范围**，避免范围蔓延：

- ❌ 交易 / 下单功能（不做券商对接）
- ❌ 复杂 K 线技术分析工具（仅 24h 迷你 K 线）
- ❌ 社交 / 评论 / 跟单
- ❌ 自训练 NER 模型（用规则词典 + 可选 LLM 回退）
- ❌ 实时 tick 级行情（用延迟行情足以支撑叙事关联）

| 指标 | 目标 |

| 脉搏页 DAU 渗透率 | > 60%（首页即脉搏，预期高） |
